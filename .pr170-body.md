# PR #170 — ToastAction internal kind + Save Retry

## What

`ToastAction` now supports a second kind besides the existing `external` (open URL):

- **`external`** — opens a URL via `window.open()` (already used by install-mode hint)
- **`internal`** — invokes a renderer-side callback, then auto-dismisses the toast

Wired `saveFailed` toasts to a "Retry" action that re-runs the current save, so users get a one-click recovery instead of having to navigate back to the toolbar.

## Why

The `ToastAction` API was defined but only the install-mode hint ever used it (and only with `external`). Error toasts are fire-and-forget — clicking the × buries them without giving users a path forward. A save/read failure is one of the few cases where the user can productively do something about the error *right now*: try again, or open the file in another app.

## Files

- **`src/components/Toast.tsx`** — `ToastAction` union now `external | internal`. External still passes `url`, internal passes `onSelect`. The action button's onClick branches on `kind`; the internal handler swallows callback errors so the toast still dismisses (user can't get stuck).
- **`src/App.tsx`** — `pushToast` signature gains an optional 3rd arg `action?: ToastAction`. Both error branches of `handleSaveCurrent` (IPC failure + `result.message ?? tf("saveFailed")`) wire a Retry action that re-invokes the current save via a `saveCurrentRef` (so the action reaches the latest closure without coupling to `activeTab`/`tf` dep churn).
- **`src/i18n.tsx`** — +2 keys (`toastActionRetry` / `toastActionDismiss`) × 2 locales = 4 strings. Match the existing 4-space indent of the toast block.
- **`src/components/Toast.test.tsx`** — new `describe("ToastStack · internal action (PR #170)")` block with 3 tests covering: onSelect invokes + auto-dismiss, no window.open side effect, swallowing callback errors.

## Why a ref instead of self-reference

`handleSaveCurrent` references itself in the Retry callback. Circular self-reference inside a `useCallback` is a TDZ trap (`Cannot access X before initialization`). The ref + `useEffect` mirror pattern lets the action handler reach the latest `handleSaveCurrent` while keeping both functions independent of each other's closure deps.

## i18n copy

- `toastActionDismiss`: 知道了 / Dismiss — zh matches informal toast register; en stays plain
