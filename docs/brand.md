# OpenMe Brand Guide

OpenMe（中文名：启吾）is a local-first file workspace platform. The brand should feel calm, precise, reliable and product-grade.

## Name

| Name | Usage |
| --- | --- |
| OpenMe | Global product name, repository name and developer-facing name. |
| 启吾 | Chinese product name for Chinese documentation, product narrative and domestic communication. |
| OpenMe 启吾 | Preferred bilingual display when the product first appears in Chinese materials. |

“启吾” means opening the file and, more importantly, opening one’s own understanding of the file. The name should be used with restraint. Do not turn it into decorative copy.

## Core line

```text
打开文件，不必先猜该用哪个软件。
Open Anything. Understand Everything.
```

## Brand origin

```text
Designed and built in Wuxi, China.
```

Use Wuxi as the product origin signature, not as a slogan replacement. The product should remain global, developer-friendly and industry-neutral.

## Voice

OpenMe should sound:

- clear
- restrained
- technical when necessary
- honest about boundaries
- local-first by default
- calm rather than playful

Avoid:

- emoji
- exaggerated claims
- vague format support promises
- decorative Chinese styling without product meaning
- claiming AutoCAD-level DWG fidelity
- claiming all media codecs are supported

## Cultural vocabulary

Use Chinese cultural language through operating principles, not ornament.

| Word | Product meaning |
| --- | --- |
| 启吾 | Open the file; open personal understanding. |
| 格物 | Understand structure, boundary and risk. |
| 开卷 | Open documents, drawings, images, books and media first. |
| 归档 | Bring scattered files back into a workspace. |
| 知新 | Turn files into summaries, evidence, checks and actions. |

### China red accent

```text
#C91F37
```

Use China red only as an accent. It should not dominate the UI.

Good uses:

- thin separators
- active state
- small highlights
- logo companion mark
- release badge accent

Avoid:

- large red backgrounds
- decorative red gradients
- aggressive alert styling
- festive visual language

## Logo usage

- README hero: 320 to 360 px wide.
- App icon: follow platform icon rules.
- Do not place the text title directly below the logo when the logo is already the visual headline.
- Keep generous whitespace around the mark.
- Do not add shadows or effects to the logo in documentation.

## README rule

The README is a product front page, not a dependency inventory.

Order:

1. Logo
2. Product line
3. Value statement
4. Quick navigation
5. Why OpenMe
6. Features
7. Architecture
8. Roadmap
9. Support boundaries
10. Documentation
11. English summary

## Claims policy

Every format claim must map to `SUPPORT_MATRIX.md` and `src/file-registry/formats.ts`.

Use:

```text
识别并尝试本地播放
安全近似预览
语义检查
外部打开
```

Do not use:

```text
完美支持
全面兼容
AutoCAD 级预览
所有格式都能打开
```
