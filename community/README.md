# OpenMe Community

OpenMe community work should be practical: issues, tests, samples, documentation and focused pull requests.

## Contribution tracks

| Track | Good first contribution |
| --- | --- |
| Format support | Add detector tests and support matrix entries. |
| Viewer quality | Fix viewer edge cases and error states. |
| Documentation | Clarify setup, support boundaries and examples. |
| Testing | Add regression cases for file detection and summaries. |
| Packs | Add safe domain-specific suggested actions. |
| Workflows | Propose explicit file-to-action workflows. |

## Maintainer expectations

Maintainers should ask for:

- clear problem statement
- before and after behavior
- test or manual verification
- support boundary updates when claims change

## Contributor expectations

Contributors should not submit:

- private customer files
- copyrighted standards or paid documents
- broad format claims without implementation
- unsafe execution behavior
- large binary fixtures without discussion

## Communication style

Be direct, technical and specific. Prefer small PRs over giant rewrites.
