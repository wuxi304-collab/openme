# OpenMe Qiwu v1.0.0

> 钢铁私塾出品 · 工业软件 + AI 工具集
> No login. Local-first. Just open the file.

First stable release of OpenMe Qiwu for Windows (x64). Two artifacts ship in this release:

| 形态 | 文件 | 大小 | 何时用 |
| --- | --- | --- | --- |
| **NSIS 安装包** | `OpenMe-Qiwu-1.0.0-setup-x64.exe` | ~110 MB | 普通装机，自动建桌面 / 开始菜单快捷方式 |
| **Portable** | `OpenMe-Qiwu-1.0.0-portable-x64.exe` | ~110 MB | 解压即用、U 盘携带、临时机器跑 |

SHA-256 in `SHA256SUMS.txt`（同目录下载）。

## 安装

### 安装包（NSIS）

1. 双击 `OpenMe-Qiwu-1.0.0-setup-x64.exe`
2. 选择安装语言（English / 简体中文）
3. 阅读并接受许可（`build/license-{en,zh}.txt`）
4. 默认装到 `%LOCALAPPDATA%\Programs\OpenMe Qiwu`
5. 勾选「创建桌面快捷方式」「创建开始菜单快捷方式」（默认勾选）
6. 安装完成即可启动

### 便携版

1. 双击 `OpenMe-Qiwu-1.0.0-portable-x64.exe`
2. 解到任意文件夹（建议新建 `OpenMe Qiwu`，文件名带版本便于日后区分）
3. 进入文件夹，双击 `OpenMe Qiwu.exe` 启动
4. 不写注册表，不写 `%APPDATA%`，适合 U 盘 / 临时机器

### 卸载

- **安装包**：控制面板 → 程序和功能 → OpenMe Qiwu → 卸载。自动清理桌面 / 开始菜单快捷方式和安装目录
- **Portable**：直接删除文件夹即可，无残留

## What's new since v0.1.0

v0.1.0 是首发，本版本是首段**系统级打磨**。重点在 chrome 体验、工业级细节、可访问性、可安装性：

### Splash screen — 从黑框到有进度

启动时不再黑屏。新版 splash 直接来自 Electron 主进程（`electron/splash/`）：

- 实时进度条 + 当前阶段文案（启动 Boot / 准备 Renderer / 加载 Assets / 准备就绪 Ready）
- 动态版本号（`app.getVersion()`，永远跟运行版本一致）
- 实时计时器（elapsed time）
- 阶段切换动画 + reduced-motion 兼容
- zh + en 双语文案

### 状态栏（StatusBar）打磨

- **字符 / 词数 chip** — `Array.from(content).length` 的 code-point 计算（CJK Extension B / 代理对都准），CJK 每字一词（与 VSCode 一致）
- **编码检测** — 自动识别 UTF-8 / UTF-8 (BOM) / UTF-16 LE / UTF-16 BE
- **行结束符** — LF / CRLF / CR
- **支持等级 chip** — 点击打开 format popover（格式详情 + 风险 + 推荐 native apps + Copy/Reveal/Open 三按钮）
- **Tab 总数 + Lines chip**，带 `role="contentinfo"` + `aria-label`
- **主题切换按钮** 接入 `aria-pressed`
- **Toast 公告** 全部走 `aria-live="polite"`

### 标签页（Tab strip）

- **HTML5 drag-to-reorder** — 红线插入指示器
- **中键关闭 / Ctrl+W**
- **键盘重排** — Ctrl+ArrowLeft / Ctrl+ArrowRight
- **Roving tabindex** + `aria-describedby`
- **未保存提示**（dirty dot + i18n aria-label）

### Sidebar 空态 + 最近文件卡片

- **空态 hero** — 主 CTA + 次要 CTA，键盘可达
- **最近文件卡片** — 点击直接打开，右键 reveal 文件位置
- **搜索框 debounce** + clear button + 结果计数

### Chrome 微动效（emil-design-eng）

- 按钮 `:active` 缩放反馈（0.92–0.97，按钮位置定）
- 列表 stagger 进入（最多 8 item，避免长列表拖沓）
- 全站 easing 统一为 `cubic-bezier(0.16, 1, 0.3, 1)`
- `prefers-reduced-motion` 全覆盖

### 复杂 Viewer 全部本地化 + 直连 CSS

PDF / Office / ZIP / Media / Font / EPUB / DWG / CAD · 每个 viewer 拆出专属 `.css`（不再散在 inline style），键盘可达 + 屏幕阅读器友好；MarkdownViewer 工具栏带 `aria-keyshortcuts`。

### i18n 全 chrome + 全 viewer 收尾

- **923 zh · 923 en keys** + ICU plural
- `audit:i18n` 全绿，hardcoded 中文 0 个
- 周知命名空间：`statusBar` / `tab` / `sidebar` / `shortcutOverlay` / `commandPalette` / 每个 viewer 自有命名空间

## 架构亮点

- **Registry 驱动**：每个文件格式在 `src/file-registry/formats.ts` 声明 extension / category / supportLevel / openStrategy，UI 自动渲染
- **诚实支持等级**：A+ / A / B / C / D / E / F 七级，D/E/F 不假装渲染，路由到 native app
- **suggestedApps**：Registry 给格式挑 native app（Photoshop / VLC / AutoCAD / Microsoft Word ...），FileSummaryPanel 和 StatusBar popover 都接上
- **Pack SDK 已预留**：工程包 / 金石包 / 账册包 / 契约包 / 典籍包 / 匠作包
- **安全边界**：安装包 / 镜像 / 模型 / ROM 默认只识别和路由，不主动执行
- **本地优先**：设置走 `localStorage` + 可选 JSON 导入导出，不上云
- **无遥测**：不收集任何行为 / 文件内容

## 已知边界

- macOS / Linux：当前只发 Windows x64；electron-builder config 已写好 mac target，可后续加
- Code signing：v1.0.0 release 是 unsigned，SmartScreen 会拦一下（点「更多信息」→「仍要运行」即可）
- 文件大小上限：默认 ~50 MB，files > 50 MB 会被建议「用专门的编辑器打开」（详见 *About* → *Runtime*）
- HEX viewer：未知 binary 始终兜底（PR #97），最大渲染 256 KiB
