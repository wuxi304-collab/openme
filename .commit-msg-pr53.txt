feat(ui): settings dialog for theme, close confirm, recent limit

Adds a Settings dialog accessible from a gear button in the title bar.
Mirrors the AboutDialog structure (ESC dismiss, overlay-click dismiss,
focus management, custom radio controls) so chrome feels consistent.

New files:
- src/settings.tsx — SettingsProvider + useSettings hook with
  localStorage persistence (key: `openme.settings.v1`). All app
  settings live in one place; the provider also applies `data-theme`
  on the document root so theme tokens re-skin the app immediately.
- src/components/SettingsDialog.{tsx,css,test.tsx} — modal dialog
  with three radio sections (theme, confirm tab close, recent files
  limit) plus reset-to-defaults and close footer actions.

Refactors:
- src/theme.tsx — ThemeProvider is now a thin wrapper that reads from
  useSettings(). The legacy `openme.theme` localStorage key is left
  orphaned (harmless). `useTheme()` API unchanged, so no consumers
  need touching.
- src/App.tsx — `recentFiles` slicing now honours
  `settings.recentLimit` (was hardcoded 50). `handleCloseTab` and
  `handleCloseAllTabs` skip the native confirm when
  `settings.confirmTabClose` is off. SettingsProvider is mounted
  outside ThemeProvider so the provider tree stays shallow.
- src/components/layout/TitleBar.tsx — adds a gear button (with
  custom 12-point SVG, mirroring the info button style) that opens
  the dialog. The button uses `settingsInfoButtonAria/Title` for
  a11y.

i18n: adds 21 zh+en `settings.*` keys (453/453 audit clean). English
phrasing reviewed: "Confirm before closing tabs", "Recent files kept",
"Reset to defaults".

Bundle: index 182.33 → 185.96 kB raw (+3.63 kB), gzip held steady at
~46 kB. All 175 tests pass (10 new SettingsDialog RTL tests). tsc
clean, audit clean, build clean.
