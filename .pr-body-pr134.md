## PR #134 — CommandPalette UX polish

**Audit-then-fix** of `src/components/CommandPalette.tsx` continuing the CSS hygiene + a11y contract test series (after #125–#133).

### Real gaps found

| Gap | Before | After |
|---|---|---|
| Footer hints unannounced to AT | `<span>↑↓ ...</span>` | `<footer role="group" aria-label="Keyboard shortcuts">` linked via `aria-describedby="command-palette-hints"` |
| Result count change is silent to SR users | nothing announced on filter | `<div role="status" aria-live="polite" aria-atomic="true">{tf(...)}</div>` announces "{count} 条结果" / "4 results" |
| Selection move is silent | `aria-selected` toggles, but no live text | second `<div aria-live="polite">` follows the selection and announces "已选中第 N 项: {label}" / "Item N of M: {label}" |
| Listbox not a focus-managed widget | role="listbox" but no activedescendant | `aria-activedescendant="command-palette-option-{id}"`, options get stable `id="command-palette-option-{id}"` |
| "4 results" needed ICU plural, was ambiguous with empty state | none | `paletteResultsAnnounce` uses ICU plural `=0 {No results} one {# result} other {# results}` / zh `0/1/` |

### Why `aria-activedescendant` not focus shift

`<button role="option">` elements stay un-focused (focus stays on the `<input>`); we just mirror which option is "active" via `aria-activedescendant`. This is the ARIA combobox/listbox pattern — focus stays on the editor so the user can keep typing without re-acquiring focus. Without `aria-activedescendant`, SRs only hear "listbox, 4 items" on filter change.

### Why TWO polite live regions (not one)

- `paletteResultsAnnounce` updates only when result count changes (filter narrow/broad)
- `paletteSelectionAnnounce` updates on every ArrowUp/Down move
- If they shared one region, SRs would announce duplicate text or interrupt themselves

### Design tokens applied

- Hint group now `<footer role="group">` instead of naked `<span>` row — semantically grouped for screen readers
- Selection announcer positioned outside the dialog so SRs announce it after dialog content

### New i18n keys (3 zh + 3 en = 6 strings)

- `paletteHintsGroupAria`: "键盘操作提示" / "Keyboard shortcuts"
- `paletteSelectionAnnounce`: "已选中第 {index} 项，共 {total} 项：{label}" / "Item {index} of {total}: {label}" (uses `tf()` for label interpolation)
- `paletteResultsAnnounce`: ICU plural `=0 {No results} one {# result} other {# results}` / "{count, plural, ...}" for zh
